# MedHack AI Assistant

Проект решает задачу бинарной классификации медицинского текста: по данным
профосмотра модель определяет, есть ли у пациента противопоказания к вредным
или опасным производственным факторам из направления.

## 🎯 Два подхода к анализу

### 1️⃣ Традиционный ML (TF-IDF + XGBoost)

- **Когда использовать:** Для быстрого инференса без GPU
- **Как запустить:** `python main.py`
- **Время ответа:** < 1 сек
- **Точность:** ~87% F1-score

**Процесс:**
- Векторизация текста TF-IDF
- XGBoost классификатор с подобранным порогом
- CSV с вердиктами

### 2️⃣ Нейросетевой LLM (Qwen2.5-7B-Instruct)

- **Когда использовать:** Для детального анализа в UI с объяснениями
- **Как запустить:** `python run_vllm_server.py` + Streamlit
- **Время ответа:** 10-30 сек (зависит от GPU)
- **Преимущества:** Логика решения, контекст, черновик документов

**Процесс:**
- Сборка структурированного промпта с контекстом
- Отправка в vLLM OpenAI API
- JSON ответ с вердиктом и обоснованием

## Текущий TF-IDF+XGBoost подход

- Используется NLP-подход на основе TF-IDF.
- В один текстовый признак объединяются поля:
  - `assigned_harmful_factors`
  - `specialist_conclusions`
- Классификатор: `XGBClassifier`
- Для дисбаланса классов используется `scale_pos_weight`
- Оптимальный порог подбирается на валидационной выборке
- После подбора модель переобучается на всей обучающей выборке

## Планируемый сценарий использования

В будущем планируется UI для врача: специалист загружает файл с данными
профосмотра, а ИИ-ассистент анализирует противопоказания и формирует черновик
документа для профпатолога.

Целевая логика продукта: построить модель, которая по данным профосмотра
определяет, к каким факторам из направления у пациента есть противопоказания.

## 🚀 Быстрый старт (Streamlit + локальная AI)

### Установка

```bash
# Установить зависимости
pip install -e .

# Установить vLLM для GPU
pip install vllm[cuda]
# или для CPU
pip install vllm[cpu]
```

### Запуск

1. **Запустите vLLM сервер** (в отдельном терминале):
```bash
python run_vllm_server.py
```

2. **Запустите Streamlit приложение** (в другом терминале):
```bash
streamlit run src/medhack_ai_assistant/streamlit_app.py
```

3. **Откроется браузер** на `http://localhost:8501`

### Проверка конфигурации

Перед запуском проверьте, что всё готово:

```bash
python check_setup.py
```

## Запуск модели (CLI)

```bash
python main.py
```

Ожидаемые входные файлы:

- `data/train.csv`
- `data/test.csv`

Результат сохраняется в `submission.csv`.

### Основной flow AI-анализа

1. **Пользователь** выбирает осмотр в Streamlit (`sections.py`)
2. **Нажимает кнопку** "Запустить AI-анализ"
3. **Pipeline** (`pipeline.py`):
   - Проверяет качество данных (Quality Gate)
   - Строит промпт с контекстом (RAG из Приказа №29н)
   - Отправляет запрос в локальную модель
4. **vLLM сервер** (`run_vllm_server.py`):
   - Использует Qwen2.5-7B-Instruct
   - Возвращает JSON с вердиктом
5. **Результат** отображается в UI (`ai_rendering.py`)
